## Source-authority clarification (locked)

- **PIAA** publishes **season W/L/total-points** only — NO per-game scores.
- **MaxPreps** publishes per-game scores (user-provided example: `maxpreps.com/games/.../...htm`).
- **PhillyLacrosse** = current source for both per-game scores AND per-player stats.

Resulting trust rule, by data type:

| Data | Authoritative source | Fallback | Last resort |
|---|---|---|---|
| Per-game score | **MaxPreps** | PhillyLacrosse | (none) |
| Season W/L | **PIAA** | computed from games | PhillyLacrosse |
| Per-player stats | **PhillyLacrosse** | (none) | (none) |

PIAA still wins where it has data; for individual game scores it has none, so MaxPreps is the operational top of the stack for the H9 reconciliation queue.

## User decisions (orchestrator-locked, see request transcript)

- (a) MaxPreps creds: **Anon scrape first**; fall back to cookie-based auth only if anon blocked. Don't store creds in code.
- (b) Auto-apply: **Yes when MaxPreps confirms a value with high confidence** (parsed integer score ≥ player-goals sum). Always write to `score_sources` audit table for reversibility.
